# 41. 缺失的第一个正数

## 题目描述

[41-缺失的第一个正数](https://leetcode.cn/problems/first-missing-positive/)

## 思路

如果没有空间复杂度的限制的话，我们可以直接用哈希表来存储数组中所有的数字，然后从最小的正数开始遍历，如果没有找到对应的数字，那么这个数字就是缺失的第一个正数。

但是题目要求空间复杂度为 O(1)，所以我们不能用哈希表。我们可以将原数组改造成哈希表的形式，来解决问题。

### 方法一

我们可以将数组中对应下标的元素是否为负数，来判断这个数字是否存在。如果对应下标的元素为负数，那么说明这个数字存在，否则就不存在，那么这个数字就是缺失的第一个正数。如果都有，那么 `n + 1` 就是缺失的第一个正数。

因此，我们要先遍历数组将负数全部改成 `n + 1`。然后，再遍历一次数组，将所有在 $[1, n]$ 范围内的数的对应下标的元素改成负数，这里需要注意，每次我们拿出数组的数时一定要采用绝对值的形式，因为这个数可能已经被改成负数了。

最后，我们再遍历一次数组，找到第一个正数的下标，这个下标就是缺失的第一个正数。

### 方法二

我们可以遍历数组，我们只关注在区间 $[1, n]$ 内的数，实际上，每个下标对应的元素与其下标的关系我们并不关注。所以，我们可以将遍历的元素和其数值相同的下标的元素交换。这样所有存在的正数其下标和其元素的数组都是一样的，我们只需要找到第一个下标和其元素数值不相等的元素即可。这个就是缺失的第一个正数。

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(1)
