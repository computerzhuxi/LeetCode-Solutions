# 1461. 检查一个字符串是否包含所有长度为 K 的二进制子串

## 题目描述

[1461-检查一个字符串是否包含所有长度为 K 的二进制子串](https://leetcode.cn/problems/check-if-a-string-contains-all-binary-codes-of-size-k/)

## 思路

### 方法一

我们可以遍历字符串 s 中所有长度为 k 的子串，并用 set 存储下来。遍历完后，只需要看看该容器是否有 2^k 项即可。
同时注意字符串 s 至少要有 2^k 个长度为 k 的字符串。

#### 复杂度

- 时间复杂度：O(k ∗∣s∣)，其中 ∣s∣ 是字符串 s 的长度。将长度为 k 的字符串加入哈希集合的时间复杂度为 O(k)，即为计算哈希值的时间
- 空间复杂度：O(k∗2^k)。哈希集合中最多有 2^k 项，每一项是一个长度为 k 的字符串。

### 方法二

每次对字符串进行操作对性能的消耗比较大，我们可以优化一下。
不同子串对于的二进制是不同的，我们可以将字串转化成整数类型来存储，可以省下许多空间。
同时，我们可以发现两个相邻的子串a和b对应的关系。
假设字串a由 s*i、s*(i+1)、……s*(i+k-1)组成，b由s*(i+1)、s*(i+2)、……s*(i+k)组成。
那么
num*a = s_i \* 2^(k-1) + s*(i+1) _ 2^(k-2) + …… + s\_(i+k-1) _ 2^0
num*b = s*(i+1) _ 2^(k-1) + s\_(i+2) _ 2^(k-2) + …… + s*(i+k) * 2^0
显然
num_b = (num_a - s_i _ 2^(k-1)) \* 2 + s_(i+k)
这样我们就不用每次都提取一串字符串了

#### 复杂度

- 时间复杂度：O(∣s∣)，其中 ∣s∣ 是字符串 s 的长度。
- 空间复杂度：O(2^k)。哈希集合中最多有 2^k 项。
