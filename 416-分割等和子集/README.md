# 416. 分割等和子集

## 题目描述

[416. 分割等和子集](https://leetcode.cn/problems/partition-equal-subset-sum/)

## 思路

1. 问题转化
题目要求判断能否将数组分割成两个和相等的子集。设数组所有元素之和为 sum，那么每个子集的和应为 target = sum / 2。于是问题等价于：能否在数组中选出一些元素，使得它们的和恰好等于 target。每个元素只能选一次，这是一个典型的 0-1 背包问题，背包容量为 target，每个元素的重量和价值均为其数值本身。

2. 快速判断不可能的情况
在开始动态规划之前，可以先进行两个剪枝判断，快速排除明显无解的情况：
   - 如果 sum 是奇数，那么 target 不是整数，直接返回 false。
   - 如果数组中的最大值大于 target，那么最大元素无论分到哪个子集，都会使该子集的和超过 target，直接返回 false。
这两个检查能过滤掉许多无效输入，避免不必要的 DP 计算。

3. 算法思路
使用一维布尔数组 dp 进行动态规划。dp[j] 表示当前已考虑的部分元素中，是否存在一个子集其元素和恰好为 j。
   - 初始化：dp[0] = true（什么都不选，和为 0）。
   - 依次遍历数组中的每个元素 num：
        - 为了确保每个元素只使用一次，内层循环从 target 向下遍历到 num（逆序）。
        - 对于每个 j，如果 dp[j - num] 为真，说明加上当前元素后可以凑出和 j，则将 dp[j] 也设为 true。
   - 逆序更新的原因是：如果正序更新，同一个 num 可能会被重复使用（例如先更新 dp[num] = true，后面更新 dp[2*num] 时又会基于这个更新，错误地认为可以拿两次）。逆序保证了 dp[j - num] 是上一轮（未考虑当前元素）的状态。
   - 遍历完所有元素后，dp[target] 的值即为答案。

## 复杂度

- 时间复杂度：O(n x target)
- 空间复杂度：O(target)
