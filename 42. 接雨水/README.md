# 42. 接雨水

## 题目描述

[42-接雨水](https://leetcode.cn/problems/trapping-rain-water/)

## 思路

这道题要求我们计算给定数组中的接雨水的总量。

$i$ 处的接水量等于两边最高的高度的较小值减去 i 处的高度。
我们可以使用两个指针，从两端出发，每次比较两个指针位置的高度并更新最高值，将两个指针位置的较小值那边的最大值减去这个较小值，就是这个位置的接水量。然后再移动这个指针。

为什么计算两边高度的较小值呢？

因为我们只移动较小值，假如左边的是较小值，那左边的最大值一定是在左指针位置的左侧或者就是该指针位置，而右指针的位置一定是跟左指针和它的左边的值进行比较过，且比左指针左边的值要大，而右边最大值一定是大于等于右指针位置的值的，也就是左边最大值一定小于右边最大值，所以该位置的接水量一定是左边最大值减去该位置的值。

假设 `leftMax` 、 `rightMax` 分别为左边最大值和右边最大值， `left` 和 `right` 分别为左指针和右指针

我们可以知道

$height[left] > height[right + 1]$ 或 $height[left - 1] < height[right]$

当 $height[left] < height[right]$:

无论是哪种情况 $leftMax \lt rightMax$

那么我们就可以得到 `left` 的接水量: $leftMax - height[left]$

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(1)
