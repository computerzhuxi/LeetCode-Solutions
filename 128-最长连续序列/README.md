# 128. 最长连续序列

## 题目描述

[128-最长连续序列](https://leetcode.cn/problems/longest-consecutive-sequence/)

## 思路

将每个数放入 set 里面去重。
对于每个数 x ，我们可以去寻找 x + 1、x + 2……直到没有下一个数为止，这样的序列就是 x 所在的连续序列。
但是这样寻找时间复杂度太高了，我们发现当我们已经把 x 的连续序列寻找完了，我们就不需要再去寻找 x + 1、x + 2…… 了，因为 x + 1、x + 2…… 都已经在之前的连续序列里面了，如果按照这个方法去找，连续序列的长度一定会小于之前的，我们可以直接跳过。
所以每个数在寻找连续序列之前，先看看 x - 1 是否存在，如果存在，就不用去寻找了。
这样我们实际上只对 set 中的数进行了一次遍历，时间复杂度为 O(n)。

## 复杂度

- 时间复杂度：O(n)
- 空间复杂度：O(n)
